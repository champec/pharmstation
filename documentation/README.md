## üìù Contributing to Documentation

When adding or updating documentation:

- Use clear, concise language
- Include relevant examples where helpful
- Link to related documents
- Keep regulatory information up-to-date
- Add sources and references where appropriate
- Follow the markdown structure used in existing docs

## üîç Documentation Standards

### File Naming
- Use lowercase with hyphens (kebab-case)
- Be descriptive: `cd-register-entry-format.md` not `format.md`

### Structure
Each documentation file should include:
- Clear title (H1)
- Brief overview/introduction
- Relevant sections (H2, H3)
- Examples where applicable
- Links to related documents
- Last updated date

### Cross-References
Link to other documents using relative paths:
```markdown
See also: [CD Schedules Explained](../pharmacy-knowledge/cd-schedules-explained.md)
```
